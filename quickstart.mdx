Google gives you instant access to the world's information through a powerful search API and a suite of cloud-based tools. Follow this guide to create your account, generate your first API key, and run a live search query — all in under ten minutes.

## Prerequisites

Before you begin, make sure you have the following:

<CardGroup cols={2}>
  <Card title="Google Account" icon="user">
    Sign up for free at [google.com](https://google.com) if you don't already have one.
  </Card>
  <Card title="API Client or Browser" icon="terminal">
    Use a modern web browser, [curl](https://curl.se/), or any HTTP client such as Postman or Insomnia.
  </Card>
</CardGroup>

## Setup Steps

<Steps>
  <Step title="Create Your Google Account">
    Navigate to [google.com](https://google.com) and click **Create account**. Fill in your name, choose an email address, and set a strong password.

    Once you submit the form, Google sends a verification email to the address you provided. Open that email and click **Verify email** to activate your account before moving on.
  </Step>

  <Step title="Generate an API Key">
    After signing in, open your account menu in the top-right corner and go to **Account Settings → API Keys**.

    Click **Create new key**, give it a descriptive name such as `my-first-key`, and click **Generate**. Copy the key immediately — Google only displays the full secret once.

    <Warning>
      Store your API key in a secure location such as an environment variable or a secrets manager. Never hard-code it in source files or commit it to version control.
    </Warning>
  </Step>

  <Step title="Make Your First API Call">
    With your key in hand, send a search request to the Google API. Replace `YOUR_API_KEY` with the key you copied in the previous step.

    <CodeGroup>
      ```bash curl
      curl -X GET "https://api.google.com/v1/search?q=hello+world" \
        -H "Authorization: Bearer YOUR_API_KEY"
      ```
    </CodeGroup>

    The `q` query parameter accepts any URL-encoded search string. The `Authorization` header authenticates your request using the Bearer token scheme.
  </Step>

  <Step title="View the Results">
    A successful request returns a JSON object containing the matched results along with relevance scores and metadata. Here is what a typical response looks like:

    ```json
    {
      "query": "hello world",
      "total": 1420000000,
      "results": [
        {
          "id": "res_01J2KX",
          "title": "Hello World — Wikipedia",
          "url": "https://en.wikipedia.org/wiki/Hello_world",
          "snippet": "A hello world program is a computer program that outputs or displays the message Hello, World!",
          "score": 0.97
        }
      ]
    }
    ```

    | Field | Type | Description |
    |---|---|---|
    | `query` | string | The search term you submitted. |
    | `total` | integer | Estimated total number of matching documents. |
    | `results` | array | Ordered list of the top matching results. |
    | `results[].id` | string | Unique identifier for the result. |
    | `results[].title` | string | Page title of the matched document. |
    | `results[].url` | string | Canonical URL of the matched document. |
    | `results[].snippet` | string | Short excerpt from the matched document. |
    | `results[].score` | float | Relevance score between 0 and 1. |
  </Step>
</Steps>
